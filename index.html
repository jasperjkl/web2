웹이란 무엇인가
Posted on 2013/12/30 by 더나은웹지기 ‘웹 개방성’, 그리고 ‘더 나은 웹’ 캠페인의 일환으로 세미나와 컨퍼런스 등의 자리에서 각계의 여러분들을 만나보면, 인터넷과 웹에 대해 관심도도 다르고, 이해도도 달라서 소통하는 데 어려움을 겪곤 합니다. ‘내 웹사이트가 어떻게 하면 웹에 더 잘 나타날 수 있는가’ 궁금해하시면서도, 실상 웹이 무엇인지, 웹 검색이 어떻게 동작하는지 잘 이해하지 못해서 무엇을 어떻게 하여야 하는지 엄두를 못 내시는 분들도 많이 봤습니다. 앞으로 이곳에 연재할 ‘웹과 웹 검색’ 시리즈에서는, 인터넷과 웹 환경을 꾸려 나가는 검색 엔지니어의 시각에서 이해하는 웹이란 무엇인지, 웹이 어떻게 진화하였고 발전하고 있는지, 웹 검색의 역할은 무엇이고 어떻게 동작하는지, 웹이 발전함에 따라 어떻게 웹 검색이 진화하고 있는지, 좀 더 알기 쉽게 설명하고자 합니다. 그리고 이 첫 글에서는 ‘웹’이란 무엇인지 한번 짚고 넘어가려 합니다. 웹페이지와 링크로 이루어진 거미줄 웹은 ‘월드 와이드 웹(World Wide Web)’의 줄임말입니다. 첫 글자를 따서 WWW라고 부르기도 합니다. ‘웹(web)’을 영어 뜻 그대로 풀이하면 ‘거미줄’이라는 뜻인데, 그렇다면 ‘월드 와이드 웹’을 직역하면 ‘세상의 크기만한 거미줄’이 되겠죠. 왜 그렇게 부르게 되었을까요? 20여 년 전, 웹의 창시자인 팀 버너스리(Tim Berners-Lee)가 처음 웹이라는 것을 만들 것을 제안할 때, 그가 ‘월드 와이드 웹’이라는 이름 대신에 고려했던 후보가 다음과 같습니다. 정보 그물망 (Information Mesh) 정보의 보고(寶庫) (The Information Mine, Mine of Information)

Hypertext Markup Language (HTML) is the standard markup language for documents designed to be displayed in a web browser. It can be assisted by technologies such as Cascading Style Sheets (CSS) and scripting languages such as JavaScript. Web browsers receive HTML documents from a web server or from local storage and render the documents into multimedia web pages. HTML describes the structure of a web page semantically and originally included cues for the appearance of the document. HTML elements are the building blocks of HTML pages. With HTML constructs, images and other objects such as interactive forms may be embedded into the rendered page. HTML provides a means to create structured documents by denoting structural semantics for text such as headings, paragraphs, lists, links, quotes and other items. HTML elements are delineated by tags, written using angle brackets. Tags such as and directly introduce content into the page.
